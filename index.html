## Site Architecture
Taking a look at your site's architecture: - URL Structure - Canonicalization - Pagination - Robots.txt - XML Sitemap - HTML Sitemap - Header Response Codes - Crawl Test - HTTP and HTTPS
### URL Structure * Don not explain this stuff too much. Explain what you need to do to manage and fix it (technical seo) - Use logical directory paths - Place blog under /blog/ A typical technical product company will have a URL structure that looks like this:
### Canonicalization - They will point out things like: - Avoid duplicate content - Canonicalization: unifying multiple pages of similar content - Redirect non-www versions to www versions or vice versa - Redirect all http traffic to https version of site
### Robots.txt - Robots.txt instructs search engines on how to crawl your site
### XML Sitemap - Sitemaps instruct search engines on which pages to index - If your site doesn't already have one create one - Create a sitemap and submit to Google and Bing - Configure your sitemap to display Posts, Pages, and other Post Types

## SERP Visibility How search engines see your site
- Uptime - Crawl Test - Header Response Codes (Redirects) - Pages Indexed (Sitemaps) + Bing? - Schema and Rich Snippets - Knowledge Graph & Sitelinks - Site Indexation Issues (Crawl Errors)
### Uptime Search engines need access to your site - Your site must be up and malware-free to be indexed - Google will blacklist malware infected sites - Intermittent uptime and extremely low response times affect bounce rate - Points for dedicated WordPress hosts - Monitor uptime with: [Pingdom](https://www.pingdom.com/), [Uptime Robot](https://uptimerobot.com/)
### Pages Indexed (Sitemaps) + Bing - Check that your pages are bing indexed - Google and Bing - Make sure they're not over-indexing your site - Back to Canonicalization and robots.txt, noindex setting
### Schema and Rich Snippets - markup site with organization and sitelinks search schema - check/test for schema markup errors - this is a large topic that can't be covered here, but you can start with an organization tag: ``` schema markup for organization - does Yoast handle it? ```
### Knowledge Graph & Sitelinks - apply organizational schema.org markup to ensure knowlege graph fully populates
### Site Indexation Issues (Crawl Errors) - check for index errors - check for extra indexing (extra domains) - want it to roughly match the amount of links in your sitemap
