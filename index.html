## On-Page SEO The HTML source code search engines will be indexing
Taking a look at your site's on-page SEO: - Meta Titles and Descriptions - Images and Alt tags - Headings and Semantic HTML
## Meta Titles and Descriptions - Describes what your page is about - Should contain your targeted (focus) keyword - Search engines will ideally use in SERPS - Make sure these are complete for every page - Can be overwritten with SEO plugin

## Images and alt tags - Assign alt text to all images - Use human readable image names - Optimize full size images or assign thumbnail sizes
## Headings and Semantic HTML - Use semantic HTML to arrange your page information - Be sure that h1 tags exist and that there is only 1 per page
## Site Architecture The technical structure of your website
Taking a look at your site's architecture: - URL Structure - Canonicalization - Robots.txt - XML Sitemap - HTML Sitemap - Crawl Test - HTTP and HTTPS
### URL Structure - Generally looking for well-optimized and logical site organization - Page heirarchy, directory structure, "siloing" - A couple quick examples:
A typical technology product company would organize products like so: ``` ## Two products https://sysdig.com/product/monitor/ https://sysdig.com/product/secure/ ## Product pricing https://sysdig.com/product/monitor/pricing/ https://sysdig.com/product/secure/pricing/ ```
Organizing "company" info: ``` https://qwilt.com/company/ https://qwilt.com/company/our-story/ https://qwilt.com/company/board/ https://qwilt.com/company/articles/ https://qwilt.com/company/press-releases/ ``` Those are some quick examples, but this is a large subject in and of itself.
### Canonicalization - Canonicalization: unifying multiple pages of similar content - Don't want multiple URLs serving the same content - This is "duplicate content", you want to avoid that - Make sure you serve a single version of your content - Redirect non-www versions to www versions or vice versa - Redirect all http traffic to https version of your site
### Robots.txt - Robots.txt instructs search engines on how to crawl your site - Make sure that you have one and it is in the right form - Use Google Seach Console to test your robots.txt file
### XML Sitemap - Sitemaps instruct search engines on which pages to index - If your site doesn't already have one create one - Create a sitemap and submit to Google and Bing - Configure your sitemap to display Posts, Pages, and other Post Types
### Crawl Test - Crawl your site as Google does through Search Console
![Search Console](images/search-console-fetch-as-google.png)
### HTTP and HTTPS - Use HTTPS. This is not negotiable. - Good hosts will provide free SSL certificates - or at least provide a simple path to it
## SERP Visibility How search engines see your site
Taking a look at your site's SERP visibility: - Uptime - Pages Indexed (Sitemaps) - Schema and Rich Snippets - Site Indexation Issues (Crawl Errors)
### Uptime Search engines need access to your site - Your site must be up and malware-free to be indexed - Google will blacklist malware infected sites - Intermittent uptime and extremely low response times affect bounce rate - Points for dedicated WordPress hosts - Monitor uptime with: [Pingdom](https://www.pingdom.com/), [Uptime Robot](https://uptimerobot.com/)
### Pages Indexed (Sitemaps) - Check that your pages are being indexed - Make sure they're not over-indexing your site - Back to Canonicalization and robots.txt, noindex setting - You can 'Disallow' or noindex Categories and Tags
### Schema and Rich Snippets - Add organization and sitelinks search schema - Check/test for schema markup errors - Yoast SEO plugin creates Website and Organization schema markup - This is a large topic that can't be covered here, but you can start with an organization tag:
You can paste in custom Schema markup: ``` { "@context": "http://schema.org", "@type": "Organization", "url": "https://www.kentik.com", "potentialAction": { "@type": "SearchAction", "target": "https://www.kentik.com/?s={search_term_string}", "query-input": "required name=search_term_string" }, "logo": "https://wpcdn.kentik.com/wp-content/themes/kentik/assets/img/kentik-logo.png", "contactPoint" : [ { "@type" : "ContactPoint", "telephone" : "+1 844-356-3278", "contactType" : "customer service" } ], "sameAs" : [ "https://www.facebook.com/kentiktech", "https://twitter.com/kentikinc", "https://www.linkedin.com/company/kentik"] } ```
Yoast provides Website and Organization Schema markup: ``` { "@context":"http:\/\/schema.org", "@type":"WebSite", "@id":"#website", "url":"https:\/\/www.sentient.ai\/", "name":"Sentient Technologies Holdings Limited", "potentialAction":{ "@type":"SearchAction", "target":"https:\/\/www.sentient.ai\/?s={search_term_string}", "query-input":"required name=search_term_string" } } { "@context":"http:\/\/schema.org", "@type":"Organization", "url":"https:\/\/www.sentient.ai\/", "sameAs":[ ], "@id":"#organization", "name":"Sentient Technologies", "logo":"https:\/\/www.sentient.ai\/wp-content\/uploads\/2017\/03\/logo-sentient-black-1.svg" } ```
Test using the [Structured Data Testing Tool](https://search.google.com/structured-data/testing-tool/u/0/) ![Structured Data Testing Tool](images/google-structured-data-testing-tool.png)
### Site Indexation Issues (Crawl Errors) - Use Google Seach Console and run the following reports - Google Index > Index Status - Google Index > Blocked Resources - Crawl > Crawl Errors - You want the indexed pages to roughly match the amount of links in your sitemap
## Usability Is your website genenerally navigable
Taking a look at your site's usability: - PageSpeed - Mobile Usability Test - Custom 404 page - HTML Sitemap
### PageSpeed - PageSpeed is an important ranking factor - Make it fast for both Desktop and Mobile - A subject too deep for this presentation
Here's some tools to help: - Speed Tests: [WebPagetest](http://www.webpagetest.org/), [PageSpeed Insights](https://developers.google.com/speed/pagespeed/insights/), [GTmetrix](https://gtmetrix.com/), [Pingdom](https://tools.pingdom.com/) - Content Delivery Networks (CDN): [Cloudflare](https://www.cloudflare.com/), [MaxCDN](https://www.maxcdn.com/), [Amazon CloudFront](https://aws.amazon.com/cloudfront/) - [Autoptimize](https://wordpress.org/plugins/autoptimize/) _Testing services are behind when it comes to HTTP2_
### Mobile Usability Test - Mobile usability is a ranking factor for Google - Your site needs to be usable on mobile devices - Test with Googleâ€™s [Mobile Friendly Test](https://search.google.com/test/mobile-friendly)

### Custom 404 Page - Your site should return a 404 page - Create a custom 404 page - Include navigation links and search - Options - Quality themes should come with a 404 page template - Plugins: [404page](https://wordpress.org/plugins/404page/) - Create your own custom template
### HTML Sitemap - Create and human readable HTML Sitemap - Options - Install a plugin for this - Create in pure html - Create your own template
